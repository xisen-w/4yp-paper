\chapter{Introduction}
\label{chapter:introduction}

\section{Personal Agents Are Capable---but Siloed}

The past two years have witnessed an explosion in AI agent capability. Systems such as ReAct \cite{yao2023react}, Toolformer \cite{schick2023toolformer}, and commercial products like Manus \cite{manus2025} and Devin \cite{devin2024} can now autonomously browse the web, write and debug code, manage calendars, and orchestrate multi-step workflows. These agents are rapidly evolving from reactive chatbots into persistent digital proxies with real-world authority.

Yet each agent operates in isolation. Your agent does not talk to my agent. When a founder's agent needs to share a project update with a colleague's agent, or when a researcher's agent must coordinate schedules with a collaborator's, the interaction does not happen autonomously---the human must extract the information, reformulate it, and transmit it through legacy channels. The result is a ``coordination tax'' that persists despite each individual agent being highly capable.

\textbf{Connecting personal agents would be transformative.} A network of interoperating agents can automate the coordination layer that currently consumes the majority of knowledge workers' time: scheduling meetings, sharing context, requesting information, negotiating commitments. This is what Pulse, the system described in this thesis, sets out to build---infrastructure for cross-boundary agent communication.

But the moment you connect two personal agents, you create a new problem. Each agent has deep access to its owner's data---notes, calendar, emails, health records, financial details. An external agent can ask yours anything. And there is no reliable mechanism to ensure your agent only shares what it should. This gap---between ``agents can talk'' and ``agents can talk safely''---is the subject of this thesis.

This gap is not merely an engineering oversight---it reflects a missing layer in the emerging agent stack. Frameworks like LangChain \cite{langchain2023}, AutoGen \cite{wu2023autogen}, and CrewAI provide capabilities \textit{within} an agent; communication protocols like MCP \cite{anthropic2024mcp} standardise how agents connect to tools. But neither addresses the fundamental question: \textit{what should one agent be allowed to know about another agent's owner?}

\section{The Security Problem: Why Agents Remain Siloed}

The reason agents remain siloed is not a lack of ambition but a genuine security crisis. Granting an agent deep personal context---notes, emails, calendar, long-term memory---makes it maximally useful to its owner, but maximally dangerous when exposed to external parties. Prompt injection attacks can override system instructions to exfiltrate protected data \cite{greshake2023not}. Social engineering can manipulate an agent's reasoning. Confused deputy attacks can trick an agent into executing unauthorised actions. The attack surface is vast, and the stakes are personal.

Recent large-scale evaluations have confirmed the severity. Nasr et al.\ \cite{nasr2025attacker} tested twelve published prompt injection defences under adaptive attacks and found that \textit{every defence} could be bypassed with greater than 90\% success; human red-teaming achieved 100\%. Meta's ``Rule of Two'' framework \cite{meta2025ruletwo} formalises the danger: an agent becomes critically vulnerable the moment it simultaneously processes untrusted inputs, accesses sensitive data, and can take state-changing actions. Even state-of-the-art model-level hardening (Anthropic reports a 1\% attack success rate for Claude \cite{anthropic2025promptinjection}) leaves a nonzero residual risk that is unacceptable when personal data is at stake.

\section{The Security--Utility Paradox}

The research community has responded with a wave of defences---content classifiers, adversarial training, chain-of-thought filtering, and multi-layered sanitisation pipelines \cite{google2025promptinjection, anthropic2025promptinjection}. These approaches improve the average case but share a fundamental limitation: \textbf{they trade information exchange utility for security}. Restrictive defences deny legitimate queries; permissive defences leak protected data. Manual permission management reintroduces the very coordination overhead that agents were supposed to eliminate.

This creates a paradox at the heart of agentic communication:

\begin{quote}
\textit{Maximum coordination utility requires maximum context access, but maximum context access creates maximum security vulnerability when interacting across boundaries.}
\end{quote}

Current solutions cluster at the extremes of this trade-off. Prompt-based guards offer high utility but catastrophic security. Manual access control lists offer perfect security but crippling overhead. No existing system navigates the Pareto frontier between the two. And crucially, \textbf{no existing benchmark measures both dimensions simultaneously}---agent capability benchmarks ignore security, and agent security benchmarks ignore the utility cost of their defences. Without a benchmark that captures the full trade-off, progress on this problem cannot be measured.

\section{Research Approach and Contributions}

This thesis has two halves, mirroring the two problems that arise when connecting personal agents:

\begin{enumerate}
    \item \textbf{Making agents talk: a coordination layer.} We design and implement Pulse, a personal agent coordination layer that enables cross-boundary agent communication. Pulse provides the infrastructure---share links, tool orchestration, per-relationship context scoping---that allows one person's agent to interact with another's. This is a systems contribution: we describe how to abstract a ``personal agent,'' what the communication layer looks like, and what architectural patterns make cross-boundary interaction possible.

    \item \textbf{Making agents talk safely: measuring and addressing the security--utility tradeoff.} The moment agents connect, a privacy problem emerges. We design a benchmark of 200 questions across 5 sensitivity categories, evaluated under multiple defence levels, relationship conditions, and attack modalities, with a dual-metric protocol that simultaneously measures task-completion utility and data-protection security. No prior benchmark captures this dual measurement across a defence gradient (\cref{sec:rw_benchmarks}). We then propose architectural solutions---Mountable Context Cells, Dual-Track Memory, and an Intelligent Escalation Protocol---motivated by the specific failure patterns the benchmark reveals.
\end{enumerate}

We address four research questions:

\begin{itemize}
    \item \textbf{RQ1}: What are the failure modes when AI agents communicate across trust boundaries?
    \item \textbf{RQ2}: How does the specificity of a defence policy affect both utility and security?
    \item \textbf{RQ3}: How should agent memory be redesigned for multi-party interactions?
    \item \textbf{RQ4}: Can architectural isolation (removing data from context) fundamentally outperform prompt-level defence (instructing the agent not to share)?
\end{itemize}

The remainder of this thesis is organised as follows. \Cref{chapter:related_work} reviews related work on agent capabilities, security, communication protocols, and existing benchmarks. \Cref{chapter:problem_formulation} formally defines the security--utility trade-off. \Cref{chapter:benchmark_design} presents the Pulse coordination layer and the benchmark design. \Cref{chapter:failure_cases} reports failure cases and baseline analysis. \Cref{chapter:solution_proposal} proposes architectural solutions. \Cref{chapter:experiments} evaluates these solutions. \Cref{chapter:conclusion} discusses limitations and future directions.
